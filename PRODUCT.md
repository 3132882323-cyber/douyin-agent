# 店策 Agent 产品定义（v2.7.0）

## 1. 产品定位

店策 Agent 是抖店商家和千川投手的“本地经营副驾”。它解决的不是“替用户自动点按钮”，而是把散落在抖店、罗盘和千川网页里的信息组织成可验证、可追溯的今日行动清单。

核心价值：

1. 不需要开发者 API，复用用户自己的网页登录态。
2. 不要求运营人员复制 JSON 或导出多个 CSV。
3. 建议必须附数据来源、页面类型和采集时间。
4. 默认只读和脱敏，把错误操作与隐私风险放在产品边界之外。

## 2. 目标用户

- 店铺运营：关心订单、商品、库存、售后、评价和经营趋势。
- 千川投手：关心消耗、成交、ROI、计划状态和素材表现。
- 店长/老板：需要一页看清今天的风险、机会和先后顺序。

## 3. P0 用户闭环

1. 安装扩展并启动本地 Agent。
2. 扩展识别已打开且已登录的抖店/千川页面。
3. 用户点击“同步并诊断”。
4. 系统按页面类型保存脱敏快照并计算质量分。
5. 侧边栏展示今日简报、异常、证据和下一步动作。
6. 用户回到对应后台核对并人工执行。

## 4. 安全等级

| 等级 | 示例 | 产品策略 |
| --- | --- | --- |
| L0 | 读取、统计、诊断 | 可自动执行 |
| L1 | 标题、话术、计划草稿 | 可生成和复制，不自动提交 |
| L2 | 改库存、发货、回复客户 | 必须展示前后差异并确认 |
| L3 | 改预算、佣金、资金动作 | 二次确认、额度限制、短时审批、审计 |
| L4 | 刷单、伪造交易、绕过风控 | 直接拒绝 |

v2.7.0 开放 L0，以及不会自动提交的 L1 经营建议和报告。本地 Agent 可随 Windows 登录自动启动，所有经营数据巡查仍仅在用户点击后运行；千川巡查支持多账号识别、分账号快照和用户显式选择，避免不同账号计划、视频库和直播数据互相覆盖。账号识别或全店巡查失败时，用户也可明确选择“当前千川页面”模式，仅采集当前激活的千川标签页并直接用于分析。千川入口失效或浏览器出现 `ERR_FAILED` 时会尝试安全回退，并将登录失效与网络加载失败明确区分；后台巡查标签页意外关闭时自动重建并重试当前页面。投放侧升级为千川投手今日工作台，每条计划建议均提供发现、判断、动作、调整范围、观察窗口和验收指标，并接入任务状态闭环。

## 5. 首版验收标准

- 扩展可在 Manifest V3 Chrome 中从解压目录安装。
- 权限不包含 `<all_urls>`、cookies 或 debugger。
- 手动同步确实触发所有已打开目标页面采集。
- 同一平台不同页面类型不会互相覆盖。
- 快照包含 schema 版本、来源、页面类型、时间、质量和隐私状态。
- 手机号、证件号、邮箱和订单编号默认脱敏；姓名、地址、账号等敏感列默认排除，隐私模式下页面全文不落盘。
- Companion Service 只监听 127.0.0.1，异常输入不会破坏已有快照。
- 侧边栏在无数据、服务未启动、数据过期和正常状态下都有明确恢复提示。
- 所有诊断包含 evidence 或明确标注为数据覆盖建议。
- 千川建议必须包含计划、消耗/ROI 等依据、建议幅度、置信度和执行护栏。
- 库存预警区分缺货、极低库存、低库存和预计可售天数。
- 日报按本机时间生成，保存在本地并支持保留天数清理。

## 6. 路线图

### P1：稳定数据模型

- 为订单、商品、售后、计划和报表建立页面 fixture 与选择器回归测试。
- 增加日期范围、店铺/账户识别和跨日快照。
- 支持导出日报与“建议有用/无用”反馈。

### P2：草稿型 Agent

- 从商品页生成标题、卖点和客服话术草稿。
- 对千川调整建议增加多日趋势、素材维度和用户采纳反馈。
- 引入审批单、差异预览和审计记录。

### P3：受控执行

- 仅对明确页面和动作开放 L2/L3 执行。
- 执行前重新读取页面、校验目标、幂等键和审批时效。
- 资金动作默认关闭，必须单独启用。
