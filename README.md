# 店策 Agent：抖店 × 千川经营副驾

一个本地优先的 Chrome 扩展和 AI Agent 数据桥。它复用你已经登录的抖店、巨量千川网页，不需要申请开发者 API，也不会读取 Cookie。

## 当前版本能做什么

- 识别已打开的抖店和千川后台页面。
- 一键同步当前可见的经营指标和表格。
- 按平台和页面类型分别保存快照，不再互相覆盖。
- 默认脱敏手机号、身份证号、邮箱和订单编号；姓名、地址、账号等敏感表格列不采集，页面全文不落盘。
- 在 Chrome 侧边栏生成今日经营简报、数据覆盖和优先处理建议。
- 以运营总管视角统一排序货架、直播、投放和库存任务，并给出负责人、依据和验收标准。
- 支持总管、货架、直播、投放和商品岗位视图；首页只展示最多 3 个必做事项，并记录待处理、进行中、待观察和已完成状态。
- 无需官方 API，可用一个后台巡检标签页依次采集经营、订单、商品、库存、售后、评价、货架、直播、短视频、图文、搜索、推荐卡、资金和千川核心页面；仅在用户点击按钮后运行，不执行后台定时巡检。
- 自动巡检逐页记录成功、失败、质量和页面类型，失败页面重试一次；完成后关闭巡检标签页并自动生成最新经营报告。
- 巡检专用页面会尝试读取虚拟滚动列表并自动采集最多 5 页分页数据，同时记录读取行数、滚动次数和分页截断提示；不会操作用户原有标签页。
- 每次脱敏快照进入本地历史库，默认保留 30 天；侧边栏展示近 7 天指标变化，为调整前后对比和自动验收提供基础。
- 分析货架曝光→点击→成交漏斗，以及店铺直播/千川直播大屏的观看→商品点击→成交漏斗。
- 按计划明细生成千川止损、保持观察、素材优化和谨慎放量建议，并展示判断依据与安全边界。
- 按缺货、极低库存、低库存和预计可售天数生成库存预警。
- 每天按设定时间自动生成本地 Markdown 经营报告，也可在侧边栏立即生成。
- 通过 MCP 向本机 AI 客户端暴露脱敏快照和诊断结果。

首版严格只读：不会修改库存、发货、回复客户、调整预算或启停计划。

## 产品架构

```text
抖店 / 千川已登录网页
        │
        ▼
Chrome 扩展（采集、脱敏、状态、侧边栏）
        │  仅 http://127.0.0.1:8765
        ▼
本地 Companion Service（分区缓存、简报、诊断）
        │
        ├── Chrome 侧边栏 /insights
        └── MCP stdio（给 Codex、Claude Desktop、Cherry Studio 等）
```

## 安装 Chrome 扩展

Chrome 不允许普通网页直接安装未上架的扩展，开发版请使用“加载已解压的扩展程序”：

1. 打开 `chrome://extensions/`。
2. 开启右上角“开发者模式”。
3. 点击“加载已解压的扩展程序”。
4. 选择本项目的 `extension` 文件夹；不要选择仓库根目录或 zip 文件。
5. 将“店策 Agent”固定到工具栏。

安装成功后，扩展详情应显示版本 `2.5.1`。

## 启动本地 Agent

双击：

```text
bridge/start_bridge.bat
```

浏览器访问 <http://127.0.0.1:8765/health>，看到 `"status": "ok"` 即表示正常。

## 日常使用

1. 在 Chrome 中登录并打开需要读取的抖店/千川页面。
2. 点击扩展图标，确认两个数据源和本地 Agent 状态。
3. 点击“立即同步”。如果页面提示采集脚本未就绪，刷新该后台页面后再同步。
4. 点击“打开经营副驾”，查看千川调整、库存预警、今日简报和数据覆盖。
5. 在“策略阈值与日报”中设置 ROI 目标、消耗判断门槛、低库存阈值和日报时间。

要同时覆盖订单、商品、售后、千川计划和报表，需要分别打开这些页面。扩展会按 `source + page_type` 保存，因此各类快照不会覆盖。

## 隐私与安全

- 扩展权限仅覆盖抖店、千川、精选联盟和 `127.0.0.1:8765`。
- 默认脱敏手机号、身份证号、邮箱和订单编号；姓名、地址、账号等敏感表格列不采集，隐私模式下页面全文不落盘，整页原始 DOM 不会发送。
- URL 会移除查询参数和 hash，避免保存账户或追踪参数。
- 本地服务只监听 `127.0.0.1`，请求体限制为 2 MB，快照采用原子写入。
- MCP 默认不返回 `page_text`；需要时必须显式传入 `include_page_text=true`。
- `bridge/data` 是本机运行数据，可能包含经营信息，不应打包或提交到代码仓库。

## 已知限制

- 网页字段没有官方稳定契约，平台改版后可能需要更新选择器。
- 虚拟滚动表格只能读取当前已渲染的行。
- 登录过期、验证码和风控必须由用户手动处理。
- 定时同步需要 Chrome 保持运行；它不是官方实时推送 API。
- 定时日报需要本地 Agent 保持运行；若 Chrome 未运行，报告会使用最近一次已缓存数据并标注数据新鲜度。
- 当前诊断是确定性规则，所有经营建议都应回到后台口径核对后再执行。
